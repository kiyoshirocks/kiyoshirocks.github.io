Learning Ruby

13 May 2018

學習Ruby on Rails的歷程

最近幾年決定重新學著程式開發，就選定 Ruby on Rails 與 JavaScript 。買書回來進修，而且看了好多年菜市場阿龍哥的文章之後，發現龍哥竟然出實體書了，可以預訂的那天就訂了一本。

對我來說，看了菜市場龍哥「為你自己學 Ruby on Rails 」一書，以及參加了五倍紅寶石的全端進階衝刺班，JavaScript 的基礎概念總算適當整合了，同時對於程式設計中一些底層運作有概念，透過理解 JavaScript，更理解了Ruby，才真正體會為什麼人家說寫 Ruby 會有幸福感。

樂透機的體悟

在初學一段時間後，模擬：「讓49個號碼先做亂序處理之後，再取出不重複的六個來，然後依序排列取出的六個號碼」的樂透開彩流程。

初學一段時間，認知中JavaScript 的其中一種方法：

// my lottery in JavaScript

var lottery = [],
    number; 

while (lottery.length < 6) { 
    number = parseInt(Math.random() * 49 + 1, 10);
    if (lottery.indexOf(number) === -1) { 
        lottery.push(number); 
    };
};
console.log(lottery);

lottery.sort(function(a, b){ return x - y });
console.log(lottery);

也有想過透過shuffle處理之後取前面六個就不會重複啦

Ruby 我能想到的是這樣完成：

# my lottery in Ruby

lottery = [*1..49].sample(6) 
# 連 [*1..49].shuffle 都省略了⋯⋯

p lottery
p lottery.sort

讀寫上面都很輕鬆、優雅、寫意。

依照鴨子型別的說法：叫起來是鴨子叫，飛起來是鴨子飛，走起來是鴨子走，他就是鴨子了。他們都能做一到四十九的亂數，也能從裡面挑出不重複的六個，又能將它們排序，於是他們都一樣是樂透機了。（笑）

Ruby依照預設的全域方法實作，常用的、語意化的方法已經先行為你實作，什麼資料結構、演算法模型、物件導向啥的，暫時不了解也沒關係，先入門就能做些東西出來了，你看樂透機功能不就這樣用一行程式碼做好，等你把結果印出來。搭配模板適當排版也一樣可以製作簡單大方的WebUI，而且實際可以用。

（演算法模型、物件導向實踐是好東西，記得要補上去。）

JavaScript的sort方法預設將數字強制型轉為字串，以字元Ansi碼作為排序依據，會有數字排序的問題，實作上就得加上搭配對於Math.random的了解與排序演算法的實作，加上強制型轉是個強大且任性的預設功能，搞定這些東西其實對於程式設計的內功也有不同的扎實，各有不同好處。

繼續用Ruby開發

體驗後覺得 Ruby 用於網路開發很方便寫意，同時學HTML5標準包含的JavaScript，前後端內外功都能互補，Rails 也能快速開發商用功能，火速上線。然後接觸到 Sinatra，一樣有著 Ruby 的特點，架構輕卻有著不同的適用情境。

Ruby、Rails、Sinatra 這些工具，效能不是頂尖但是夠用，效能頂尖的語言與框架，也是寫得出沒效率的Code，終究是基本功的問題。僅僅只會九種語言的加減乘除，用在程式開發當然是不夠用的，練好基本功當然比起用什麼語言重要，而且精通了一種程式語言，學其他的程式語言，進入的速度就會快很多。

Sinatra + JavaScript是我的進階目標，現在就快點做好可以實際應用的 side-project 吧！
